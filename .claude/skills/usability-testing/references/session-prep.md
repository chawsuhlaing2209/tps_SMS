# Session Prep Reference

## Moderator Guide Template

### Opening script (read verbatim or close to it)

> "Thanks for joining today. We're testing a design — not your skills. There are no wrong answers. If something is confusing, that's valuable feedback for us, not a reflection on you.
>
> I'll ask you to think out loud as you work — narrate what you're looking at, what you expect to happen, and what you're trying to do. I won't be able to answer questions during tasks, but I'll be here the whole time.
>
> We'll be recording the session. The recording stays internal and is only used by our design team. Is that okay with you?"

---

### Warm-up questions (5 min)

Ask 2–3 of these based on user type:

- "Tell me a bit about your role and what you use [system type] for day to day."
- "How comfortable are you with [booking systems / dashboards / this type of tool]?"
- "Walk me through the last time you had to [relevant task context]."

Goal: calibrate their mental model before they touch anything.

---

### Think-aloud prompts (use during tasks)

| Situation | Prompt |
|-----------|--------|
| User goes silent | "What are you thinking right now?" |
| User hesitates on an element | "What do you expect this to do?" |
| User makes an error | "What did you expect to happen there?" |
| User succeeds quickly | "How confident are you that worked?" |
| User asks if they're doing it right | "What would you do if I weren't here?" |

Never: hint at the answer, confirm they're right, or say "good job" during a task.

---

### Post-task questions (after each task)

1. "How would you rate that task on a scale of 1–7?" (Single Ease Question — 1 = very difficult, 7 = very easy)
2. "What was the most confusing part?"
3. "Was there anything you expected to see that wasn't there?"

---

### Post-session debrief (5–10 min)

- "If you could change one thing about what you saw today, what would it be?"
- "Was there anything that worked really well that you'd want to keep?"
- "Any questions for me now that we're done?"

## Observer Note-Taking Template

| Time | User action | Quote / verbalization | Observation type |
|------|------------|----------------------|-----------------|
| 0:42 | Clicked wrong tab first | "I thought the bookings would be here" | Navigation confusion |
| 1:15 | Scrolled past CTA, paused | "Where do I save this?" | CTA visibility issue |

**Observation types:** Navigation error / Confusion / Task failure / Recovery / Expectation mismatch / Positive reaction / Suggestion

## Consent Form (Short Version)

**Usability Study Participant Consent**

I agree to participate in a usability study conducted by [Company/Team]. I understand:

- The session will be recorded (screen + audio/video)
- Recordings are used only for internal design research
- My participation is voluntary and I can stop at any time
- My name will not be attached to any published findings

## Environment Checklist

**Remote sessions:**
- [ ] Zoom / Meet link tested
- [ ] Screen share permissions confirmed
- [ ] Recording enabled and tested
- [ ] Prototype accessible via link (no login required)
- [ ] Backup: screenshot deck ready if prototype fails

**In-person sessions:**
- [ ] Device matches target user's device type
- [ ] Screen recording software running
- [ ] Separate observer space set up (they must not be visible to participant)
- [ ] Consent form printed or on tablet

## Session Timing Template (60 min)

| Time | Activity |
|------|----------|
| 0–5 min | Welcome, consent, explain think-aloud |
| 5–10 min | Warm-up questions |
| 10–45 min | Tasks (3–5 tasks, ~7 min each) |
| 45–55 min | Post-session debrief |
| 55–60 min | Wrap-up, thank participant |
